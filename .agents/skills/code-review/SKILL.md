# Skill: Code-Review
- **Aktivierung:** "Reviewe meinen Code", "Prüfe diesen Code", "Ist dieser Rust-Code idiomatisch?".
- **Kriterien:**
  1. Idiomatik (`match`, `if let`, `.unwrap_or()`, `Option`, `Result`).
  2. Ownership & Borrowing (keine unnötigen `.clone()`, Scopes/Referenzen optimal).
  3. Formatierung (`rustfmt`, `snake_case`).
- **Didaktik:** **Keine fertigen Lösungen!** Nur Tipps, Leitfragen, Pseudocode oder Compiler-Fehler erklären.
