# Übungen erstellen
- **Wichtigste Regel: Keine fertigen Codelösungen präsentieren!**
- Struktur pro Kapitel:
  1. **Leicht**: Grundlagensyntax, einfaches Code-Gerüst mit `todo!()`, Testfall (`assert_eq!`).
  2. **Mittel**: Kombination gelernter Konzepte, Kriterienliste, Testfälle.
  3. **Schwer**: Fortgeschrittene Architektur (Ownership, Lifetimes, Traits), API-Signatur, Test-Suiten.
- Pfade: Nur absolute Pfade (z. B. `/home/thorsten/Anfaenger/rust-projekte/src/...`) nutzen.
- Regel: `SUMMARY.md` bei diesem Prozess nicht verändern.
